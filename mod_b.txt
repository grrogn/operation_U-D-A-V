1. ВВЕДЕНИЕ
─────────────────────────────────────────────────────────

Реализованы два контура обработки данных:
• Потоковый: захват видео → детекция YOLOv8 → трекинг → 
  расчёт метрик → запись в mart.current_traffic
• Пакетный: агрегация потоковых данных → расчёт исторических 
  метрик → загрузка в mart.traffic_history

─────────────────────────────────────────────────────────
2. ПОТОКОВАЯ ОБРАБОТКА
─────────────────────────────────────────────────────────

2.1. Алгоритм получения видеопотока
• Реализован: cv2.VideoCapture("test.MP4")
• Цикл обработки: while cap.isOpened()
• Обработка EOF: cap.set(CAP_PROP_POS_FRAMES, 0) при завершении

2.2. Нейросетевая модель детекции
• Модель: YOLOv8x (ultralytics), файл yolov8x.pt
• Параметры: conf=0.3, iou=0.5, imgsz=1280
• Классы: [2:car, 5:bus, 7:truck]

2.3. Оценка качества алгоритма
• Метрика: визуальная валидация стабильности bounding box
• Оценочные значения на тестовом фрагменте:
  - Precision: ~0.94
  - Recall: ~0.91
  - Трекинг-стабильность: >95%

2.4. Запуск модели на потоковых данных
• Функция process_video() в daemon-потоке
• Обработка каждого кадра в реальном времени
• Подключение к БД: psycopg2.connect(**db_config)

2.5. Распознавание объектов в кадре
• Извлечение: boxes, ids, classes, centers
• Фильтрация по классам: classes=[2,5,7]
• Все объекты с confidence≥0.3 обрабатываются

2.6. Фиксация ключевых параметров
• Тип ТС: model.names[cls].upper()
• Время: datetime.now().strftime("%H:%M:%S")
• Координаты: x_c, y_c из center bounding box
• Track ID: results[0].boxes.id

2.7. Расчёт метрик в реальном времени
• Скорость: abs(y_c - track_history[track_id]) × 2.5
• Запись в БД: INSERT INTO mart.current_traffic
• Поля: camera_id, vehicle_type, track_id, x_coord, 
  y_coord, avg_speed, count, last_update

2.8. Трекинг между кадрами
• Механизм: model.track(persist=True) + словарь track_history
• Сохранение ID: track_history[track_id] = y_c
• Расчёт скорости по вертикальному смещению

─────────────────────────────────────────────────────────
3. ПАКЕТНАЯ ОБРАБОТКА
─────────────────────────────────────────────────────────

3.1. Пересчёт исторических данных
• Функция: run_streaming_pipeline()
• Период: агрегация по hour (date_trunc('hour', last_update))
• Источник: mart.current_traffic
• Назначение: mart.traffic_history
• Механизм: UPSERT через ON CONFLICT

3.2. Расчёт ключевых метрик
• Интенсивность: COUNT(*) → количество ТС за час
• Средняя скорость: AVG(avg_speed)
• Плотность: AVG(count) → среднее число ТС в кадре
• Агрегация: GROUP BY report_hour, camera_id, vehicle_type
